Search For Creative Images

Use this endpoint to search our contemporary stock photos, illustrations and archival images.

Quickstart

You'll need an API key and access token to use this resource. Please see our Getting Started page for more information on how to sign up for an API key. There are several SDKs available for popular programming languages to help you get started.

Use the following endpoint to search for creative images:

GET https://api.gettyimages.com:443/v3/search/images/creative?phrase=<query term>

Add your API key to the request header:

Api-Key: <your api key>

Your request would look like this using curl:

curl -X GET -H "Api-Key: <your api key>" https://api.gettyimages.com/v3/search/images/creative?phrase=<query term>

The response body will look like the following example: 

{
  "result_count": 1,
  "images": [
    {
      "id": "<image id}",
      "asset_family": "creative",
      "caption": "<image description>",
      "collection_code": "<collection code>",
      "collection_id": <collection id>,
      "collection_name": "<collection name>",
      "display_sizes": [
        {
          "is_watermarked": false,
          "name": "thumb",
          "uri": "<display url>"
        }
      ],
      "license_model": "rightsmanaged",
      "max_dimensions": {
        "height": 2738,
        "width": 4442
      },
      "title": "<image title>"
    }
  ]
}

You can show different information in the response by specifying values on the "fields" parameter (see details below). You can search with only an API key, and that will give you search results that are equivalent to doing a search on the GettyImages.com site without being logged in (anonymous search). If you are a Getty Images API customer and would like to ensure that your API searches return only assets that you have a license to use, you need to also include an authorization token in the header of your request. Please consult our Authorization FAQ for more information on authorization tokens, and our Authorization Workflows for code examples of getting a token. To include your API token in the search request, add it to the headers as a Bearer token (example in curl):

-H "Authorization: Bearer {your-token}"

Http Request Headers

Api-Key string Api-Key Required.
Accept-Language string Accept-Language parameter optional. Used to restrict the set of natural languages that are preferred as a response to the request.
Authorization string Authorization token optional. Used for applying content access restrictions. Use one of the OAuth2 grants to auto-populate this value.

Sorting and Pagination

Clients can use the request parameters “page” and “page_size” to support paging through the set of matching images returned in a response.

page integer Page number.
page_size integer Size of page.
sort_order string Allows sorting of results. Possible values: best_match most_popular newest

Fields

The fields parameter is used to change the shape of the response returned from a search query. Specify the names of the specific fields that you want to see in your response. Fields parameter values ending in “_set” do not represent a single field, but are a shorthand for returning a related set of commonly used fields. The API will process your request most efficiently if you specify only the fields that you need. "keywords" and "largest_downloads" especially consume additional resources so only specify them if you specifically need them.

You'll need a Resource Owner Grant token to include the largest_downloads in your response.

allowed_use Communicates licensing information to customers who make one-off image purchases. Most API customers access images through their agreements and do not need to consult this field.
alternative_ids Identifies the synonym ids for the asset.
artist Identifies the creator of the asset.
asset_family Specifies the asset family filter to apply to the search results. The possible values are: Creative Editorial
call_for_image Indicates that you must call your Getty Images sales rep to obtain access to high res files for this image. This image is not available for online download.
caption A brief description of the asset.
collection_code A three-letter code identifying the collection.
collection_id Identifies the collection to which the asset belongs.
collection_name Identifies the name of the collection to which the asset belongs.
color_type Indicates if the asset is color, or black and white. The possible values are: Color Black/White
comp Large display size; sized for layout composition. Comp images are larger than preview and thumbnail, but not full sized, averaging about 600 pixels on the longest side.
copyright Identifies asset copyright.
date_created Identifies the creation date of the asset.
detail_set A collection of metadata fields commonly used for displaying information in a search results based experience.
display_set A collection of metadata fields that includes detailed information about each asset.
graphical_style Indicates the graphic style of the asset. The possible values are: Photograph Illustration
id A unique numerical string that identifies the asset.
keywords Identifies the type of keyword. The possible values are: Unknown Specific people Ethnicity Age Gender Number of people Concept Location Viewpoint Image technique Composition Color Additional info Candidate term Fashion phrases Entertainment
largest_downloads Returns hypermedia download links to the largest size.
license_model Identifies a licensing model for the asset. The possible values are: RoyaltyFree RightsManaged
max_dimensions Pixel dimensions of the largest available file.
orientation Specifies orientation of assets returned in search results. The possible values are: Horizontal Vertical PanoramicHorizontal PanoramicVertical Square
prestige Indicates if this asset is part of the Prestige collection.
preview Medium display size; sized for popup previews. Preview images are larger than thumbnail but smaller than comp, and are about 270 px on the longest side.
product_types Identifies a customer’s product offering that authorizes access to an asset. The possible values are: easyaccess editorialsubscription premiumaccess royaltyfreesubscription imagepack
quality_rank Indicates the image quality. Possible values are 0 - 4.
referral_destinations Contains a ReferralDestination entry for each of the sites on which the asset can be found.
summary_set A collection of metadata fields commonly used for displaying information in a search results based experience.
thumb Smallest display size; sized for a results grid. Thumbnail images are smaller than preview and comp, and have a dimension of 170 pixels on the longest side.
title Indicates the title of the asset.
uri_oembed Identifies a URI that can be used to get oEmbed data for the asset. Please note that this will only be populated for embeddable assets. See our oEmbed documentation for more information.

Fieldsets

Fieldsets are a shorthand for returning a related set of commonly used fields.

summary_set asset_family caption collection_id collection_code license_model max_dimensions thumb title
display_set preview comp thumb high_res_comp
detail_set allowed_use artist call_for_image caption collection_id collection_code collection_name copyright date_created editorial_segments event_ids graphical_style asset_family license_model max_dimensions orientation referral_destinations title preview product_types quality_rank thumb comp

Filters

Filters provide a mechanism for narrowing down the set of matching images to only those images meeting certain criteria. At least one query parameter must be provided in addition to the selected filters.

You'll need a Resource Owner Grant token to filter by product_types.

age_of_people array Allows for filtering based on age of individuals in an image Possible values: newborn baby child teenager young_adult adult adults_only mature_adult senior_adult 0-1_months 2-5_months 6-11_months 12-17_months 18-23_months 2-3_years 4-5_years 6-7_years 8-9_years 10-11_years 12-13_years 14-15_years 16-17_years 18-19_years 20-24_years 20-29_years 25-29_years 30-34_years 30-39_years 35-39_years 40-44_years 40-49_years 45-49_years 50-54_years 50-59_years 55-59_years 60-64_years 60-69_years 65-69_years 70-79_years 80-89_years 90_plus_years 100_over
artists string Free-text search, comma-separated artist search query.
collection_codes array Allows filtering by collection code.
collections_filter_type string Allows searching based on included or excluded collections. Possible values: include exclude
compositions array Allows for filtering based on image composition Possible values: abstract candid close_up copy_space cut_out full_frame full_length headshot looking_at_camera macro portrait sparse still_life three_quarter_length waist_up
embed_content_only boolean Restrict search results to embeddable images. The default is false.
ethnicity array Filters search results based on the ethnicity of individuals in an image. Possible values: black caucasian east_asian hispanic_latino japanese middle_eastern mixed_race_person multiethnic_group native_american_first_nations pacific_islander south_asian southeast_asian
exclude_nudity boolean Exclude images tagged as containing nudity. The default is false.
file_types array Identifies a file type for the image. Possible values: eps gif jpg png
graphical_styles array Identifies a graphical style for the image. Possible values: fine_art illustration photography
keyword_ids array Comma separated-list of keyword IDs. If specified, returns images containing the requested keyword(s). If keyword IDs and phrase are both specified, returns only those images matching the query phrase which also contain the requested keyword(s).
license_models array Identifies a licensing model for the image. Possible values are: royaltyfree, rightsmanaged Possible values: rightsmanaged royaltyfree
number_of_people array Allows for filtering based on the number of people in the image Possible values: none one two group
orientations array Choose images based on their aspect ratios. Possible values: Horizontal Vertical Square PanoramicHorizontal PanoramicVertical
phrase string Free-text search query.
prestige_content_only boolean Restrict search results to prestige images. The default is false.
product_types array Allows filtering by a list of product types. Possible values: easyaccess editorialsubscription imagepack premiumaccess royaltyfreesubscription

Errors

400 InvalidParameterValue Received when an invalid querystring parameter is passed or an unknown value is specified for an enumerated type.
400 UnknownProviderUri Received when an unknown provider uri is passed to entity_uris.
400 IncorrectlyFormedUri Received when an incorrectly formatted provider uri is passed to entity_uris.
400 MalformedRequest Received when a potentially malicious input string is received from the client as part of the request data.
400 InvalidPage Received if page is less than zero or greater than the number of pages in the search results.
400 InvalidPageSize Received if page_size is greater than 100.
401 AuthorizationTokenRequired Received when largest_download is added to the fields query string parameter and the Authorization header is missing or contains an invalid token.
403 UnauthorizedDisplaySize Received when high_res_comp is requested but not authorized for the given authorization token.
403 NoAccessToProductType Received when product_types is requested but not authorized for the given authorization token.
